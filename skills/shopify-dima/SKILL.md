# Shopify DIMA

## Overview

Shopify DIMA turns store data into operator-ready growth actions. DIMA means **Diagnose, Instrument, Move fast, Automate**: identify the current constraint, define how it will be measured, ship the highest-confidence reversible changes, then systematize what works.

Use the supporting handover for deeper examples: `references/shopify-dima-handover.md`.

## Required Intake

Ask for missing inputs when possible. If the request is urgent, state assumptions and data gaps before recommending action.

- Goal, store stage, category, margin constraints, and decision owner.
- Comparison window and baseline: week-over-week, month-over-month, campaign period, or launch period.
- Sessions, conversion rate, AOV, revenue, orders, gross margin or contribution margin if available.
- Checkout, cart, product-page, and landing-page drop-off.
- Top products by revenue, margin, traffic, conversion, inventory status, and return rate.
- Channel mix, ad spend, MER/ROAS/CAC, UTMs, and landing pages.
- Email/SMS list size, flow revenue, campaign calendar, and segmentation.
- Recent changes to theme, apps, pricing, promos, shipping, inventory, or ad creative.

## DIMA Flow

1. **Diagnose** - Locate the constraint in the KPI tree: profitable revenue -> traffic, conversion, AOV, margin, and retention. Segment by channel, device, product, landing page, new/returning customer, and funnel step.
2. **Instrument** - Confirm the measurement path before prescribing tactics. Name the primary KPI, guardrails, baseline, owner, data source, launch date, and decision rule.
3. **Move fast** - Prioritize reversible changes with clear upside. Use: `priority = impact x confidence x speed x measurability / risk`. Pick one main bet and one or two quick wins for the week.
4. **Automate** - Turn repeated wins into flows, templates, dashboards, checklists, saved segments, recurring QA, or default operating procedures.

## Output Contract

Every DIMA handoff must include:

1. **Situation and constraint** - One paragraph naming what is holding growth back.
2. **KPI read** - Current numbers, baseline comparison, and explicit data gaps.
3. **Ranked action plan** - Table with rank, action, DIMA rationale, owner, KPI, guardrail, effort/risk, and decision date.
4. **Weekly experiment loop** - What ships now, what gets measured, and the Friday scale/iterate/kill/hold decision.
5. **Founder/operator asks** - Only the decisions or assets needed to unblock execution.
6. **Automation follow-up** - What to codify if the experiment works.

## Common Mistakes

| Mistake | Correction |
| --- | --- |
| Guessing a different DIMA acronym | Use Diagnose, Instrument, Move fast, Automate. |
| Listing tactics without a constraint | Start with the KPI tree and name the bottleneck. |
| Optimizing revenue while margin worsens | Prefer contribution profit or profitable revenue as the north star. |
| Shipping changes without a decision rule | Define baseline, KPI, guardrail, and decision date first. |
| Asking the founder to approve every detail | Escalate only brand, offer, budget, and irreversible decisions. |
